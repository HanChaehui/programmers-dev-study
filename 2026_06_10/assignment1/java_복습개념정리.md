
인스턴스 변수 = 참조형이므로 해당 인스턴스의 시작 주소 저장, 함수 파라미터로 넘기면 함수 내부에서 값을 수정할 수 있음, C/C++ 처럼 const로 수정 막는 문법이 없고 final로만 수정 제한 가능

new로 생성하여 복사 → deep copy

new로 생성하지 않고 선언과 동시에 이미 생성된 인스턴스 복사 → Shallow copy

## 오버로딩 Overloading

- 자바에서 메서드를 유연하고 효율적으로 사용하는 중요한 기법
- 동일한 작업을 수행하되, **입력 형태에 따라 다른 동작**을 요구할 때 사용
- 같은 이름의 메서드를 여러 개 정의할 수 있게 해주는 자바의 기능
- 단, 이 메서드들은 매개변수의 타입, 갯수, 또는 순서가 달라야 함
- 메서드 오버로딩은 **같은 기능을 수행하지만 입력이 다른 경우**에 사용되며, 이를 통해 코드의 가독성과 유지보수성을 높일 수 있음
    
    `public int add(int… numbers)` → 가변인자, 0개 이상
    

#### 오버로딩의 규칙

1. 매개변수의 타입 : 매개변수의 데이터 타입이 다르면 오버로딩이 가능
2. 매개변수의 개수 : 매개변수의 개수가 다르면 오버로딩이 가능
3. 매개변수의 순서 : 매개변수의 타입이 다를 경우, 그 순서를 바꿔서 오버로딩 가능

*주의 : 반환 타입이 다르다고 오버로딩이 성립하는 것은 아님, 즉 매개변수의 목록이 동일하지만 반환 타입만 다른 메서드는 오버로딩으로 간주되지 않으며, 컴파일 오류 발생

#### 오버로딩의 장점

- 코드의 가독성 : 같은 이름의 메서드를 사용해 **다양한 입력**을 처리할 수 있어 코드가 직관적
- 유지보수 용이성 : 관련된 작업을 수행하는 메서드를 **같은 이름으로 그룹화**하여 코드를 유지 관리하기 쉬움
- 다형성 : 컴파일 시점에서 적절한 메서드를 선택하는 **컴파일 타임 다형성**을 제공

## 생성자 Constructor

- 반환 타입이 없으며, **객체가 생성될 때 자동으로 호출**되는 특수한 메서드
- 객체의 **초기 상태를 설정**하는 데에 사용되며, **클래스 이름과 동일한 이름**을 가져야함

#### 생성자의 주요 특징

1. **클래스 이름과 동일** : 생성자는 클래스 이름과 동일한 이름을 가져야함
2. **반환 타입 없음** : 생성자는 반환 타입을 명시하지 않음. 반환 타입을 명시하면 일반 메서드로 인식함
3. **오버로딩 가능** : 생성자는 메서드와 마찬가지로 오버로딩이 가능, 즉 매개변수의 개수나 타입이 다른 **여러 개의 생성자를 정의할 수 있음**
4. **기본 생성자 자동 추가** : 클래스에 생성자를 명시하지 않으면 컴파일러가 매개변수가 없는 기본 생성자를 자동으로 추가함, 하지만 사용자가 하나라도 생성자를 정의하면 컴파일러는 기본 생성자를 자동으로 추가하지 않음
5. **객체 초기화** : 생성자는 객체의 필드를 초기화하는 데에 사용됨

#### 생성자의 종류

1. 기본 생성자
    - 매개변수가 없는 생성자
    - 클래스에 생성자가 정의되지 않은 경우, 컴파일러가 자동으로 제공하는 생성자
        
        ```java
        		// 기본 생성자
            public F_person1() {
                System.out.println("기본 생성자");
                // 초기화
                name = "Paul";
                age = 20;
            }
        
        				...
        				// 기본 생성자를 사용하여 객체 생성
                F_person1 person1 = new F_person1();
                person1.display();
        ```
        
2. 매개변수가 있는 생성자
    - 객체를 생성할 때 인자를 전달받아 필드를 초기화하는 생성자
    - 여러 개의 생성자를 오버로딩하여 다양한 초기화 방법을 제공할 수 있다.
        
        ```java
        		// 매개변수가 있는 생성자
            public F_person2(String name, int age) {
                // name, age -> 지역변수
                // 초기화
                this.name = name;
                this.age = 30;
            }
        
        				...
        				// 매개변수가 있는 생성자를 사용하여 객체 생성
                F_person2 person2 = new F_person2("Allice");
                person2.display();
        
        ```
        
3. 복사 생성자
    - **동일한 클래스의 다른 객체**를 복사하여 새로운 객체를 생성하는 생성자
    
    ```java
    		// 복사 생성자
        public F_person4(F_person4 other) {
            this.name = other.name;
            this.age = other.age;
        }
    
    				...
    				// 원본 객체
            F_person4 person4 = new F_person4("Charlie", 25);
    
            // 복사 생성자를 사용하여 객체 복사
            F_person4 person4_1 = new F_person4(person4);
    ```
    
4. 다른 생성자를 호출하는 생성자
    - 생성자의 이름으로 클래스 대신 this를 사용
    - 자바 구 버전에서는 한 생성자에서 다른 생성자를 호출할 때는 반드시 첫 줄에서만 호출이 가능 (java 25 최신 버전에서는 가능)
        
        ```java
        		// 다른 생성자를 호출하는 기본 생성자
            public F_person5() {
        		    // F_person5("John", 25); 에러
        		    this("John", 25);
        		}
        				
        				// 생성자에서 다른 생성자 호출
        				F_person5 person5 = new F_person5();
        ```
        

#### 생성자의 역할과 이점

1. 객체 초기화
2. 코드의 가독성
3. 오버로딩을 통한 유연성
4. 캡슐화 : 생성자를 통해 필수적인 초기화가 강제되므로, 객체가 항상 유효한 상태로 생성될 수 있음

## 상속 Inheritance

- 하나의 클래스가 다른 클래스의 속성과 메서드를 물려받아 사용하는 것을 의미함
- 상속을 통해 코드 **재사용성**을 높이고, **계층 구조**를 형성하며, 기존 클래스를 **확장**하여 새로운 클래스를 정의할 수 있음

**#상속 #재정의 #확장**

#### 주요 특징

1. **부모 클래스(슈퍼 클래스)**
    
    상속을 제공하는 클래스, 다른 클래스들이 상속받을 수 있는 일반적인 속성과 메서드를 정의함
    
2. **자식 클래스(서브 클래스)**
    
    부모 클래스를 상속받아 확장하는 클래스, **부모 클래스의 모든 속성과 메서드를 물려받으며** 필요에 따라 추가적인 속성이나 메서드를 정의할 수 있음
    
3. **`extends` 키워드**
    
    자식 클래스가 부모 클래스를 상속받기 위해 사용
    
4. **메서드 오버라이딩** `@Override`  ( ↔ 오버로딩 )
    
    이때 부모 클래스의 메서드와 **동일한 이름, 반환 타입, 매개변수**를 가져야함
    (+ 부모 클래스의 멤버변수와 같은 이름의 멤버변수 중복 정의도 가능)
    
5. 다형성 Polymorphism
    
    상속을 통해 자식 클래스는 부모 클래스의 형태로 사용될 수 있으며, 이는 **코드의 유연성**과 **확장성**을 높임
    
6. 단일 상속
    
    자바는 ‘다중 상속’을 허용하지 않고 ‘단일 상속’만 허용함 ( ↔ C++)
    

## 오버라이딩 Overriding (재정의)

- 상속받은 부모 클래스의 **메서드를 자식 클래스에서 재정의**하여 사용하는 기법
- 오버라이딩을 통해 자식 클래스는 부모 클래스의 기본 기능을 유지하면서도, 자신만의 방식으로 그 기능을 수정하거나 확장할 수 있음

#### 오버라이딩 조건

- 자식 클래스에서 오버라이딩하는 메서드는 부모 클래스의 메서드와 이름, 매개변수가 같아야함
- 단, 접근 제어자와 예외는 제한된 조건 하에서만 다르게 변경할 수 있음
    1. 접근 제어자는 부모 클래스의 메서드보다 좁은 범위로 변경 불가능, 
    (만일 부모 클래스에 정의된 메서드의 접근 제어자가 `protected`라면 이를 오버라이딩하는 자식 클래스의 메서드는 접근 제어자가 `protected`나 `public`이어야 함)
        
        *접근 제어자 [`public` → `protected` → (`default`) → `private`] 접근 범위 넓은순
        
    2. 부모 클래스의 메서드보다 많은 수의 예외를 선언할 수 없음

## Object 클래스 - 모든 클래스의 조상

- 모든 클래스 상속 계층도의 최상위에 있는 조상 클래스
- 컴파일러는 자동으로 상속이 없는 클래스에 `extends Object`를 추가함으로써 모든 클래스의 조상이 되도록 함
- 따라서 모든 클래스들은 Object 클래스의 멤버들을 상속받으므로 Object 클래스에 정의된 멤버들을 사용할 수 있음 ex) toString(), equals()

## `this`, `super` 참조변수

`this` **: 현재 객체 자신**을 가리키는 참조변수

주로 인스턴스 변수와 매개변수 또는 지역변수의 이름이 같을 때, 이를 구분하기 위해 사용

`super` : **자식 클래스에서 조상 클래스의 멤버를 참조할 때** 사용하는 참조변수

조상 클래스에서 상속받은 멤버와 자식 클래스에서 새로 정의한 멤버의 이름이 같을 때, `super`를 사용하여 조상 클래스의 멤버를 구분할 수 있음

- 조상 클래스로부터 상속받은 멤버도 결국 자식 객체의 일부이므로, 이름이 중복되지 않는 경우에는 `this`로도 접근할 수 있음
- 따라서 `super`는 보통 조상 클래스의 멤버와 자식 클래스의 멤버를 명확히 구분해야 할 때 사용하는 것이 좋음
- `this`와 `super`는 모두 현재 객체와 관련된 참조이지만, `this`는 현재 클래스의 멤버를 기준으로 접근하고, `super`는 조상 클래스의 멤버를 기준으로 접근한다는 차이가 있음
- `super`는 별도의 조상 객체를 가리키는 것이 아니라, 현재 객체 안에 포함된 조상 클래스 영역의 멤버에 접근하기 위한 키워드임
- `this`와 `super`는 인스턴스와 관련된 키워드이므로, 객체 생성 없이 호출되는 **`static` 메서드에서는 사용할 수 없음**
- 인스턴스 메서드가 호출될 때, 현재 객체를 가리키는 참조가 내부적으로 전달되고, 그 참조를 `this`로 사용할 수 있음

#### `super()` - 조상 클래스의 생성자

- `this()`와 `super()`는 모두 생성자 안에서 다른 생성자를 호출할 때 사용
- `this()`는 **같은 클래스의 다른 생성자**를 호출할 때 사용
- `super()`는 **조상 클래스의 생성자**를 호출할 때 사용
- 자식 클래스의 인스턴스를 생성하면, 자식 클래스의 멤버와 조상 클래스의 멤버가 함께 포함된 하나의 객체가 생성
- 따라서 자식 객체가 조상 클래스의 멤버를 사용할 수 있으려면, **조상 클래스의 멤버들이 먼저 올바르게 초기화**되어야 함
- **자식 클래스의 생성자에서는 반드시 조상 클래스의 생성자를 호출해야 함**
- 조상 클래스의 생성자가 먼저 호출되어야 하므로, **`super()`는 생성자의 첫 줄에 작성**해야 한다.
- 생성자의 첫 줄에는 `this()` 또는 `super()` 둘 중 하나만 올 수 있으며 `this()`를 호출한 경우에는, 호출된 다른 생성자 안에서 결국 `super()`가 호출됨
- 생성자 호출은 상속 관계를 따라 계속 올라가며 반복되고, 마지막에는 모든 클래스의 최고 조상인 `Object` 클래스의 생성자까지 호출됨
- 따라서 `Object` 클래스를 제외한 모든 클래스의 생성자는 첫 줄에서 반드시 같은 클래스의 다른 생성자(`this()`) 또는 조상 클래스의 생성자(`super()`)를 호출해야 함
- 만약 생성자 첫 줄에 `this()`나 `super()`를 직접 작성하지 않으면, 컴파일러가 자동으로 `super()`를 추가함
- **단, 조상 클래스에 기본 생성자 `super()`가 없고 매개변수가 있는 생성자만 존재한다면, 자식 클래스 생성자에서 `super(값)` 형태로 직접 호출해야 함**

```java
class Parent {
    Parent(String name) {
        System.out.println("Parent 생성자");
    }
}

class Child extends Parent {
    Child() {
        super("홍길동"); // 부모 클래스에 기본 생성자가 없으므로 직접 호출해야 함
        System.out.println("Child 생성자");
    }
}
```

`super("홍길동")`을 안 쓰면 컴파일러가 자동으로 `super()`를 넣으려고 하는데, `Parent`에는 `Parent()` 생성자가 없어서 오류

## 접근 제어자

- 접근 제어자는 **클래스, 멤버변수, 메서드, 생성자**에 사용되며, 외부에서 해당 클래스나 멤버에 접근할 수 있는 범위를 제한하는 역할
- 접근 제어자를 사용하면 객체의 내부 구현을 숨기고, 필요한 부분만 외부에 공개할 수 있음
1. `public` : 모든 클래스에서 접근 가능
2. `protected` : 같은 패키지 내에서 접근 가능, 
                        다른 패키지에서는 자손 클래스만 접근 가능
3. `(default)` : 같은 패키지 내에서만 접근 가능,
                         아무 접근 제어자도 붙이지 않으면 자동으로 `(default)` 
4. `private` : 같은 클래스 내에서만 접근 가능

#### 접근 범위 순서

`public > protected > default > private`

#### 주의할 점

- 클래스에는 `public`과 `(default)`만 사용할 수 있다.
- 멤버변수, 메서드, 생성자에는 `public`, `protected`, `(default)`, `private`을 모두 사용할 수 있다.
- `private` 멤버는 외부에서 직접 접근할 수 없으므로, 보통 `getter`와 `setter` 메서드를 통해 간접적으로 접근한다.

## 캡슐화 Encapsulation

**“필드는 숨기고, 메서드로 접근하게 해서 데이터를 안전하게 보호하는 것”**

- 캡슐화는 객체지향 프로그래밍의 기본 원칙 중 하나
- 객체의 데이터를 외부에서 직접 접근하지 못하도록 숨기고, 필요한 기능만 메서드를 통해 제공하는 방식
- 즉, 객체의 내부 상태를 보호하고, 외부에서는 정해진 방법으로만 객체를 사용할 수 있게 하는 개념

#### 캡슐화의 목적

- 외부에서 객체의 필드를 함부로 변경하지 못하게 함
- 잘못된 값이 저장되는 것을 막아 데이터의 무결성을 유지
- 객체 내부 구현을 숨겨 코드의 보안성과 유지보수성을 높임
- 내부 코드가 변경되더라도 외부에서 사용하는 방식은 크게 바뀌지 않도록 함

#### 캡슐화 구현 방법

- 멤버 변수는 보통 `private`으로 선언하여 외부에서 직접 접근하지 못하게 한다.
- 필요한 경우 `getter`와 `setter` 메서드를 제공하여 값을 읽거나 변경할 수 있게 한다.
- **`private` 멤버 변수** : 클래스 외부에서 직접 접근 불가
- **`setter` 메서드** : 외부에서 private 멤버 변수 값을 변경할 수 있도록 제공, 유효성 검사를 추가하여 올바른 값만 저장되도록 할 수 있음
- **`getter` 메서드** : 외부에서 private 멤버 변수 값을 가져올 수 있도록 제공

#### 캡슐화의 주요 특징

1. **데이터 은닉**
    - 객체의 필드를 `private`으로 선언하여 외부에서 직접 접근하지 못하게 한다.
    - 이를 통해 객체의 상태가 무분별하게 변경되는 것을 막을 수 있다.
2. **접근 제어자 사용**
    - `private`, `protected`, `public` 등의 접근 제어자를 사용하여 클래스 멤버에 대한 접근 범위를 제한한다.
    - 캡슐화에서는 보통 필드는 `private`, 외부에 공개할 메서드는 `public`으로 선언한다.
3. **Getter와 Setter 사용**
    - 외부에서 필요한 경우 `getter`와 `setter`를 통해 필드에 간접적으로 접근한다.
    - `setter`를 사용하면 값 변경 전에 유효성 검사를 할 수 있다.
