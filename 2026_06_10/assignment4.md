# CPU 스케줄링

CPU 스케줄링은 여러 프로세스에게 CPU 사용 순서를 정해 주고, CPU를 공정하고 효율적으로 배분하는 작업이다.

모든 프로세스는 CPU를 사용하고 싶어 한다. 하지만 CPU는 한 번에 하나의 작업만 처리할 수 있기 때문에, 운영체제는 어떤 프로세스에게 먼저 CPU를 줄지 결정해야 한다.

이처럼 프로세스들에게 CPU를 배분하는 방식을 **CPU 스케줄링**이라고 한다.

---

# 프로세스 우선순위

**프로세스마다 중요도와 실행 특성이 다르다**.

운영체제는 각 프로세스에 **우선순위**를 부여하고, 우선순위가 높은 프로세스를 먼저 실행하거나 더 자주 실행되도록 관리한다.

우선순위가 높은 프로세스란 일반적으로 더 빨리 처리해야 하는 프로세스를 의미한다.

대표적으로 다음과 같은 프로세스의 우선순위가 높을 수 있다.

- 입출력 작업이 많은 프로세스
- 실시간 처리가 필요한 프로세스
- 일부 중요한 백그라운드 프로세스

운영체제는 프로세스의 PCB에 우선순위 정보를 저장하고, 이 우선순위를 참고하여 CPU를 배분한다.

---

# CPU 버스트와 입출력 버스트

대부분의 프로세스는 CPU만 사용하는 것이 아니라, CPU 작업과 입출력 작업을 반복하며 실행된다.

예를 들어 워드 프로세서는 다음과 같은 과정을 반복한다.

1. CPU를 사용하여 명령어를 실행한다.
2. 키보드 입력을 기다린다.
3. 입력된 내용을 저장한다.
4. 화면에 내용을 출력한다.
5. 다시 CPU를 사용하여 명령어를 실행한다.

이처럼 프로세스는 일반적으로 **CPU 사용 구간**과 **입출력 대기 구간**을 반복한다.

## CPU 버스트

CPU 버스트는 프로세스가 CPU를 사용하여 명령어를 실행하는 구간이다.

즉, 실제 계산이나 명령어 처리가 이루어지는 시간이다.

## 입출력 버스트

입출력 버스트는 프로세스가 입출력 작업이 끝나기를 기다리는 구간이다.

예를 들어 키보드 입력, 파일 저장, 화면 출력, 디스크 접근 등을 기다리는 시간이 여기에 해당한다.

정리하면 다음과 같다.

| 구분 | 의미 |
| --- | --- |
| CPU 버스트 | CPU를 사용하여 명령어를 실행하는 구간 |
| 입출력 버스트 | 입출력 작업이 끝나기를 기다리는 구간 |

---

# CPU 집중 프로세스와 입출력 집중 프로세스

프로세스는 CPU를 많이 사용하는지, 입출력 작업을 많이 하는지에 따라 크게 두 가지로 나눌 수 있다.

## CPU 집중 프로세스

CPU 집중 프로세스는 입출력 작업보다 CPU 작업이 많은 프로세스이다.

즉, CPU 버스트가 긴 프로세스이다.

예시는 다음과 같다.

- 복잡한 수학 연산
- 컴파일 작업
- 그래픽 처리
- 대용량 데이터 계산

CPU 집중 프로세스는 대기 상태보다 실행 상태에 오래 머무르는 경향이 있다.

## 입출력 집중 프로세스

입출력 집중 프로세스는 CPU 작업보다 입출력 작업이 많은 프로세스이다.

즉, 입출력 버스트가 많은 프로세스이다.

예시는 다음과 같다.

- 비디오 재생
- 파일 다운로드
- 디스크 백업
- 키보드 입력 처리
- 화면 출력 작업

입출력 집중 프로세스는 실행 상태보다 대기 상태에 오래 머무르는 경향이 있다.

---

# 입출력 집중 프로세스를 먼저 실행하는 이유

CPU 집중 프로세스와 입출력 집중 프로세스가 동시에 CPU를 요구한다면, 입출력 집중 프로세스를 먼저 실행하는 것이 더 효율적일 수 있다.

입출력 집중 프로세스는 CPU를 오래 사용하지 않고, 잠깐 실행된 뒤 곧바로 입출력 작업을 기다리는 대기 상태가 되는 경우가 많다.

따라서 입출력 집중 프로세스를 먼저 실행하면 다음과 같은 장점이 있다.

1. 입출력 작업을 빠르게 시작할 수 있다.
2. 입출력 장치가 쉬지 않고 동작할 수 있다.
3. 입출력 집중 프로세스가 대기 상태가 된 동안 다른 프로세스가 CPU를 사용할 수 있다.
4. CPU와 입출력 장치를 더 효율적으로 사용할 수 있다.

즉, 입출력 집중 프로세스를 먼저 처리하면 CPU와 입출력 장치가 동시에 효율적으로 사용될 수 있다.

# 우선순위 기반 CPU 배분

운영체제는 프로세스마다 우선순위를 부여한다.

그리고 PCB에 저장된 우선순위 정보를 바탕으로 어떤 프로세스를 먼저 실행할지 결정한다.

우선순위가 높은 프로세스는 일반적으로 더 빨리 실행되거나, 더 자주 CPU를 할당받는다.

예를 들어 다음과 같은 프로세스들이 있다고 하자.

| PID | 우선순위 |
| --- | --- |
| 012 | 10 |
| 123 | 2 |
| 456 | 5 |

만약 우선순위 숫자가 작을수록 우선순위가 높다고 가정하면, 실행 순서는 다음과 같다.

```
123번 프로세스 → 456번 프로세스 → 012번 프로세스
```

## 정리

CPU 스케줄링 : 여러 프로세스에게 CPU를 어떤 순서로 배분할지 결정하는 작업

운영체제는 프로세스의 우선순위, CPU 사용량, 입출력 작업 여부 등을 고려하여 CPU를 효율적으로 배분한다.
