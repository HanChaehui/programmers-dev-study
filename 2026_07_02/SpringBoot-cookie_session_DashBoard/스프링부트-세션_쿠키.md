# 세션 + 쿠키로 개인화 대시보드 만들기 (로그인은 세션, 취향은 쿠키)

> 수업에서 배운 **세션**과 **쿠키**를 한 화면에서 함께 써 봐요. 둘의 **역할이 왜 다른지**를 코드로 직접 느끼는 게 목표예요.
> - **세션** = 로그인 상태(누구인지) → 민감하고 잠깐만 필요 → **서버**가 기억
> - **쿠키** = 테마·마지막 방문 시각 → 안 민감하고 오래 유지 → **브라우저**가 기억
>
> 프론트는 타임리프로 **아주 간단하게** 만들어요. 화면 꾸미기보다 "어디에 무엇을 저장하느냐"에 집중해요.
>
> 💡 각 Step의 **힌트는 접혀 있어요.** "이건 세션이 맞나, 쿠키가 맞나?"를 먼저 고민하고 막히면 펼치세요.

---

## 0. 먼저 알아둘 점

- 완성하면 이렇게 동작해요: 로그인하면 대시보드가 열리고, **"마지막 방문: …"** 문구와 **내가 고른 테마(밝게/어둡게)**가 보여요. **로그아웃하면 로그인 정보는 사라지지만**, 테마와 방문 기록은 **그대로 남아** 있어요. 이 차이가 이번 과제의 핵심이에요.
- 로그인 안 한 사람이 대시보드 주소로 바로 들어오면 **로그인 페이지로 되돌려** 보내요(세션 기반 접근 제어).
- 스프링 부트 3 기준이라 서블릿 관련 import는 `jakarta.servlet.http.*` 예요. (부트 2면 `javax...`)
- ⚠️ **쿠키 값 주의**: 쿠키 값에는 공백·콜론 같은 문자를 그냥 넣기 어려워요. 그래서 "마지막 방문 시각"은 **숫자(밀리초)로 저장**하고, 화면에 보여줄 때만 사람이 읽는 형식으로 바꿔요.

## 1. 무엇을 만드나요?

| 화면(주소) | 하는 일 | 세션? 쿠키? |
|------|------|------|
| `GET /login` | 로그인 폼 | (이미 로그인 상태면 대시보드로) |
| `POST /login` | 이름을 **세션**에 저장 | 세션 |
| `GET /dashboard` | 로그인한 사람만. 환영 문구 + 마지막 방문 + 테마 | 세션(접근 제어) + 쿠키(방문/테마) |
| `GET /theme?mode=...` | 테마를 **쿠키**에 저장 | 쿠키 |
| `GET /logout` | 세션 비우기 | 세션 |

**대시보드 화면 예시(두 번째 방문, 다크 테마)**
```
환영합니다, kim 님!
마지막 방문: 2026-06-24 08:43:15
현재 테마: dark   [밝게] [어둡게]
[로그아웃]
```

## 3. 핵심 개념

### (1) 세션 vs 쿠키 한눈에
| 구분 | 세션(Session) | 쿠키(Cookie) |
|------|------|------|
| 저장 위치 | **서버** | **브라우저** |
| 브라우저는 뭘 갖나 | `JSESSIONID`(세션 식별표)만 | 실제 이름=값 전부 |
| 수명 | 기본: 브라우저 종료/타임아웃까지 | `maxAge`로 내가 지정 |
| 보안 | 값이 서버에 있어 상대적으로 안전 | 값이 브라우저에 노출 → 민감정보 부적합 |
| 어울리는 용도 | **로그인 상태** 등 민감·일시적 정보 | **테마·최근 본 상품** 등 비민감·지속 정보 |


### (2) 쿠키 읽기: 수업의 `getCookies()` 루프 → `@CookieValue`
```java
Cookie[] cookies = request.getCookies();      // 전체를 받아서
for (Cookie c : cookies) {                     // 원하는 이름을
    if ("theme".equals(c.getName())) { ... }   // 직접 찾기
}
```
스프링은 이걸 한 줄로 줄여줘요. **같은 일**이에요.
```java
@CookieValue(value = "theme", defaultValue = "light") String theme
// "theme" 쿠키가 있으면 그 값을, 없으면 "light"를 넣어 준다.
```

```
[기억법] 로그인=세션(서버가 기억) / 테마·방문기록=쿠키(브라우저가 기억) · 없으면 리다이렉트로 문지기
```

## 5. Step by Step

### Step 1. 로그인 — 이름을 세션에 저장 (`LoginController`, `login.html`)

**목표**: 폼으로 받은 이름을 **세션에 저장**하고, 저장됐으면 대시보드로 보내요.

**할 일**
1. `GET /login`: 로그인 폼(`login.html`)을 보여줘요. 단, **이미 세션에 username이 있으면** 대시보드로 리다이렉트해요.
2. `POST /login`: `username`을 세션에 저장하고 `redirect:/dashboard`.
3. `login.html`: 이름 입력 + 제출 폼.

<details>
<summary>💡 힌트 보기</summary>

```java
import jakarta.servlet.http.HttpSession;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.*;

@Controller
public class LoginController {

    @GetMapping("/login")
    public String loginForm(HttpSession session) {
        // 이미 로그인 상태면 폼을 다시 보여줄 필요 없이 대시보드로
        if (session.getAttribute("username") != null) {
            return "redirect:/dashboard";
        }
        return "login";
    }

    @PostMapping("/login")
    public String login(@RequestParam String username, HttpSession session) {
        session.setAttribute("username", username);  // 이 순간부터 '로그인된 상태'
        return "redirect:/dashboard";
    }
}
```

```html
<!-- templates/login.html -->
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head><meta charset="UTF-8"><title>로그인</title></head>
<body>
  <h1>로그인</h1>
  <!-- th:action="@{/login}" 로 컨텍스트 경로까지 맞춰 준다 -->
  <form th:action="@{/login}" method="post">
    <input type="text" name="username" placeholder="이름을 입력하세요" required>
    <button type="submit">로그인</button>
  </form>
</body>
</html>
```

`redirect:`가 핵심이에요. POST로 저장한 뒤 곧바로 화면을 그리지 않고 **GET /dashboard로 다시 요청**하게 만들어요(새로고침해도 폼이 재전송되지 않는 안전한 패턴, PRG: Post-Redirect-Get).

</details>

**확인**: 이름을 넣고 로그인하면 `/dashboard`로 이동하면 돼요. (대시보드는 다음 Step에서 만들어요. 아직 에러 나도 정상이에요.)

### Step 2. 대시보드 — 로그인 안 하면 못 들어오게 (`DashboardController`)

**목표**: 대시보드는 **로그인한 사람만** 볼 수 있어요. 세션에 username이 없으면 로그인 페이지로 되돌려 보내요.

**할 일**
1. `GET /dashboard`: 세션에서 username을 꺼내요.
2. **없으면** `redirect:/login`. **있으면** 모델에 담아 `dashboard` 화면으로.

<details>
<summary>💡 힌트 보기</summary>

```java
import jakarta.servlet.http.HttpSession;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class DashboardController {

    @GetMapping("/dashboard")
    public String dashboard(HttpSession session, Model model) {
        String username = (String) session.getAttribute("username");

        // 세션 기반 접근 제어: 로그인 안 했으면 들여보내지 않는다
        if (username == null) {
            return "redirect:/login";
        }

        model.addAttribute("username", username);
        return "dashboard";
    }
}
```

이게 "문지기"예요. 로그인이라는 **일시적·민감한 상태**는 서버(세션)가 들고 있으니, 서버가 직접 통과 여부를 판단할 수 있어요. (쿠키에 "로그인됨=true"를 넣으면 사용자가 조작할 수 있어 위험해요 — 그래서 로그인은 세션!)

</details>

**확인**: 로그인하고 들어가면 대시보드가 보이고, **로그아웃 상태로 `/dashboard`에 직접 접속하면** 로그인 페이지로 튕겨야 해요.

### Step 3. 쿠키로 "마지막 방문 시각" 기억하기 (`DashboardController`)

**목표**: 대시보드에 올 때마다 ① 지난번 방문 시각을 **쿠키에서 읽어** 보여 주고, ② 이번 방문 시각으로 **쿠키를 갱신**해요.

**할 일**
1. `@CookieValue(value="lastVisit", required=false)`로 지난 방문 값(밀리초 문자열)을 받아요. 첫 방문이면 null.
2. null이 아니면 사람이 읽는 시각으로 바꿔 모델에 담아요.
3. 지금 시각(`System.currentTimeMillis()`)을 문자열로 만들어 `lastVisit` 쿠키에 저장(응답에 추가)해요.

<details>
<summary>💡 힌트 보기</summary>

```java
import jakarta.servlet.http.*;
import org.springframework.web.bind.annotation.CookieValue;
import java.time.*;
import java.time.format.DateTimeFormatter;

private static final DateTimeFormatter FMT =
        DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");

@GetMapping("/dashboard")
public String dashboard(
        HttpSession session,
        @CookieValue(value = "lastVisit", required = false) String lastVisit, // 없으면 null
        HttpServletResponse response,
        Model model
) {
    String username = (String) session.getAttribute("username");
    if (username == null) return "redirect:/login";
    model.addAttribute("username", username);

    // ① 지난 방문 시각(밀리초 문자열)을 사람이 읽는 형식으로 변환
    if (lastVisit != null) {
        long millis = Long.parseLong(lastVisit);
        String readable = Instant.ofEpochMilli(millis)
                .atZone(ZoneId.systemDefault())
                .format(FMT);
        model.addAttribute("lastVisit", readable);
    }

    // ② 이번 방문 시각을 쿠키에 갱신 (다음에 오면 '마지막 방문'으로 보인다)
    Cookie visit = new Cookie("lastVisit", String.valueOf(System.currentTimeMillis()));
    visit.setMaxAge(30 * 24 * 60 * 60); // 30일
    visit.setPath("/");
    visit.setHttpOnly(true);
    response.addCookie(visit);

    return "dashboard";
}
```

왜 시각을 **숫자(밀리초)로** 저장할까요? 쿠키 값엔 공백·콜론 같은 문자를 그냥 못 넣어서예요. `2026-06-24 08:43:15`엔 공백과 콜론이 있죠. 그래서 저장은 숫자로, **표시할 때만** 사람이 읽는 형식으로 바꿔요.

</details>

**확인**: 대시보드를 **두 번** 열어 보세요. 첫 방문엔 "마지막 방문"이 안 뜨고, 두 번째부터 직전 시각이 떠야 해요. 개발자도구(F12) → Application → Cookies 에서 `lastVisit`가 보이면 성공이에요.

### Step 4. 쿠키로 테마(밝게/어둡게) 유지하기 (`DashboardController` + `dashboard.html`)

**목표**: 사용자가 고른 테마를 **쿠키**에 저장하고, 대시보드가 그 테마로 보이게 해요.

**할 일**
1. `GET /theme`: 파라미터 `mode`(light/dark)를 받아 `theme` 쿠키에 저장하고 `redirect:/dashboard`. (로그인 폼에서 쓴 `@RequestParam` 그대로예요.)
2. 대시보드에서 `@CookieValue(value="theme", defaultValue="light")`로 현재 테마를 읽어 모델에 담아요.
3. `dashboard.html`에서 테마에 따라 `<body>` 클래스를 바꾸고, 마지막 방문/환영 문구/로그아웃/테마 링크를 배치해요.

<details>
<summary>💡 힌트 보기</summary>

```java
// DashboardController 에 메서드 추가
@GetMapping("/theme")
public String setTheme(@RequestParam String mode, HttpServletResponse response) {
    // 허용된 값만 저장 (light/dark 외에는 light로)
    String value = "dark".equals(mode) ? "dark" : "light";
    Cookie theme = new Cookie("theme", value);
    theme.setMaxAge(30 * 24 * 60 * 60);
    theme.setPath("/");
    response.addCookie(theme);
    return "redirect:/dashboard";
}
```
```java
// dashboard() 파라미터에 추가 + 모델에 담기
@CookieValue(value = "theme", defaultValue = "light") String theme
// ...
model.addAttribute("theme", theme);
```
```html
<!-- templates/dashboard.html -->
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
  <meta charset="UTF-8"><title>대시보드</title>
  <style>
    body.light { background:#ffffff; color:#222222; }
    body.dark  { background:#222222; color:#eeeeee; }
    a { color:#3b82f6; }
  </style>
</head>
<!-- 쿠키에서 읽은 theme 값을 body 클래스로 붙인다 -->
<body th:classappend="${theme}">

  <h1 th:text="|환영합니다, ${username} 님!|">환영합니다</h1>

  <!-- 마지막 방문: 값이 있을 때만 표시 -->
  <p th:if="${lastVisit != null}" th:text="|마지막 방문: ${lastVisit}|">마지막 방문</p>
  <p th:if="${lastVisit == null}">첫 방문을 환영합니다!</p>

  <p>현재 테마: <b th:text="${theme}">light</b></p>
  <!-- @{/theme(mode='dark')} 는 /theme?mode=dark 링크를 만든다 (물음표 뒤 파라미터) -->
  <a th:href="@{/theme(mode='light')}">밝게</a> |
  <a th:href="@{/theme(mode='dark')}">어둡게</a>

  <hr>
  <a th:href="@{/logout}">로그아웃</a>
</body>
</html>
```

`th:classappend="${theme}"`가 `<body class="dark">`처럼 만들어 주고, 위 `<style>`의 `body.dark`가 적용돼요. `th:if`로 "마지막 방문"은 값이 있을 때만 보이게 했어요.

</details>

**확인**: [어둡게]를 누르면 화면이 어두워지고, 새로고침해도 유지돼요. 개발자도구에서 `theme=dark` 쿠키를 확인해 보세요.

### Step 5. 로그아웃 — 그리고 세션 vs 쿠키 차이 체감 (`LoginController`)

**목표**: 로그아웃으로 **세션만** 비워요. 그러면 로그인 정보는 사라지지만 **쿠키(테마·방문기록)는 남는다**는 걸 직접 확인해요.

**할 일**
1. `GET /logout`: `session.invalidate()` 후 `redirect:/login`.
2. 로그아웃한 뒤 **다시 로그인**해서, 테마와 마지막 방문이 **그대로 남아 있는지** 확인해요.

<details>
<summary>💡 힌트 보기</summary>

```java
// LoginController 에 추가
@GetMapping("/logout")
public String logout(HttpSession session) {
    session.invalidate();       // 서버가 가진 세션(=로그인 상태)을 통째로 비운다
    return "redirect:/login";
}
```

`invalidate()`는 **서버 쪽 세션**만 지워요. 브라우저에 저장된 `theme`·`lastVisit` 쿠키는 건드리지 않아요. 그래서 로그아웃 후 다시 로그인하면 테마가 그대로죠. **이게 "서버가 기억(세션)" vs "브라우저가 기억(쿠키)"의 실물 차이**예요.

</details>

**확인 (이번 과제의 하이라이트)**:
1. 다크 테마로 바꾸고 → 로그아웃 → `/dashboard` 접속하면 **로그인 페이지로 튕김**(세션이 사라졌으니까).
2. 다시 로그인하면 **여전히 다크 테마 + 지난 방문 시각**이 보임(쿠키는 살아 있으니까).
   → "왜 로그인은 사라졌는데 테마는 남았을까?"를 설명할 수 있으면 이 과제를 이해한 거예요. 🎉

- [ ] 세션과 쿠키의 저장 위치·수명·용도 차이를 표로 설명할 수 있다
- [ ] 왜 로그인은 세션에, 테마는 쿠키에 두는지 이유를 댈 수 있다
- [ ] 세션이 없을 때 리다이렉트로 접근을 막는 흐름을 짤 수 있다
- [ ] `@CookieValue`가 수업의 `getCookies()` 루프와 같은 일을 한다는 걸 안다
- [ ] `maxAge`와 쿠키 삭제(수명 0)의 원리를 안다
- [ ] 로그아웃(`invalidate`) 후에도 쿠키가 남는 이유를 설명할 수 있다

## 7. 최종 완성 체크리스트

- [ ] 로그인하면 대시보드가 열리고, 로그아웃 상태로 `/dashboard` 직접 접속 시 로그인으로 튕긴다
- [ ] 두 번째 방문부터 "마지막 방문: …"이 표시된다
- [ ] 테마를 바꾸면 유지되고, 개발자도구에서 `theme`·`lastVisit` 쿠키가 보인다
- [ ] 로그아웃 후 재로그인 시 테마·방문기록이 그대로 남아 있다
